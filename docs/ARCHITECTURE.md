# Архитектура

IRPFFmpeg состоит из двух исполняемых файлов.

## Start_IRPFFmpeg

`Start_IRPFFmpeg.exe` - небольшой загрузчик. Его задача - подготовить runtime-окружение и запустить основное приложение.

Он выполняет следующие шаги:

- определяет папку, из которой запущен;
- создает `heap_dll`, если папки нет;
- переносит DLL из корня релизной папки в `heap_dll`;
- проверяет обязательные DLL;
- добавляет `heap_dll` в `PATH` дочернего процесса;
- запускает `IRPFFmpeg.exe` с исходными аргументами командной строки.

Такой подход помогает держать корень релизной папки аккуратным и снижает количество ошибок у пользователя.

## IRPFFmpeg

`IRPFFmpeg.exe` - основное приложение Win32.

Крупные зоны ответственности:

- окно приложения и элементы управления;
- плейлист и история;
- управление потоком воспроизведения;
- декодирование и ресемплинг через FFmpeg;
- аудиообработка;
- запись MP3/FLAC;
- загрузка и кэширование обложек;
- сохранение состояния.

## Основные файлы

`IRPFFmpeg.cpp`

Главный файл приложения: Win32 UI, глобальное состояние, обработчики команд, управление воспроизведением, таймеры, трей и взаимодействие с остальными подсистемами.

`audio_dsp.cpp`

Realtime-обработка аудио: DC offset removal, Dynamic Auto Volume, DeepBass, Stereo Width, Exciter, GainRider и финальный лимитер.

`file_recording.cpp`

Запись аудио в MP3/FLAC через FFmpeg, очередь аудиочанков, формирование безопасных имен файлов, добавление метаданных и обложек.

`cover_art.cpp`

Поиск обложек, HTTP-запросы через WinINet, проверка изображений и кэширование.

`metadata_decode.cpp`

Декодирование текстовых метаданных с учетом UTF-8, Windows code pages и типичных ошибок кодировки.

`util.cpp`

Общие функции конвертации строк, работы с путями и вспомогательная логика.

## Runtime-данные

Приложение работает рядом со своими файлами и создает локальные runtime-данные:

- `app.dat` - состояние и настройки;
- `playlist.m3u` - плейлист;
- `Rec/` - записи;
- `cover_cache/` - кэш обложек;
- `debug_log.txt` - диагностический лог.

Эти файлы не предназначены для хранения в git.

# Security Policy

## Supported Versions

Пока проект не имеет стабильной схемы версионирования. Сообщайте о проблемах для последней опубликованной версии или текущей ветки разработки.

## Reporting a Vulnerability

Если вы нашли уязвимость, не публикуйте exploit в открытом issue. Создайте приватное сообщение автору репозитория или используйте GitHub Security Advisories, если они включены в репозитории.

Особенно важные зоны:

- обработка сетевых потоков;
- разбор метаданных;
- загрузка обложек;
- запись файлов на диск;
- работа с DLL рядом с приложением.
